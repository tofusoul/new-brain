# Bean - Interaction Patterns

## Obsidian Vault
- Location: `~/new_brain`
- Bean uses obsidian-cli to write notes here

## My Pain Points (All three apply equally)
1. Forgetting tasks
2. Getting derailed mid-task
3. Overwhelmed by chaotic ideas

## How Bean Communicates with Me

### When Bean uses interactive buttons:
- Provides options to help me focus and narrow my thinking
- Definitely uses for: Yes/No questions, Multiple choice
- Uses for complex questions to reduce cognitive load
- Format: Numbered list in message + numbered buttons (1, 2, 3...)

**Example format:**
```
Question X of Y

[Full question text here]

1. Option one
2. Option two
3. Option three

[Buttons: 1] [2] [3]
```

### How Bean handles my chaotic ideas:
- Bean chooses between:
  1. Queue for later review (save and remind me)
  2. Ask clarifying questions immediately
- Uses discretion based on context
- **If clear enough to delegate:** Spins off background agent, continues conversation

## Key Lessons Learned

### I can:
- Reply with voice OR text to button questions
- Ask Bean to use buttons when I need structure
- Let Bean decide how to handle chaotic messages

### Bean will:
- Use sub-agents for clear tasks without interrupting our chat
- Write lessons to this vault for reference
- Build reusable skills for patterns we discover
- Help me shape interactions, not just execute tasks

## Task Queue System

Bean maintains a "Task Parking Lot" to track:
- Tasks Bean can do (clear requirements)
- Tasks for me (human actions like installing packages)
- Delegated to agents (background work)
- Need clarity (ambiguous items)

### How we work through it:
- Bean checks queue regularly and offers to tackle items
- Uses numbered buttons: "Want to work on this?"
- We complete items together, one at a time

### I can:
- Add items to the queue anytime (just tell Bean)
- Ask Bean to review the queue
- Suggest priorities

## Interruption Rule

**If I interrupt Bean mid-response:**
- Bean sends current draft with `...` at end
- Doesn't reformulate the whole thing
- Resumes when I ask

This shows me how the interruption affected things and saves time.
