## Meeting Topics - First Meeting

### Current Build (Transactional)
- Transactional data model approach
- Benefits:
  - Log of project progress as work is done
  - No messing with already completed work
  - Clean analysis without data crowding
  - Normalized way to handle different data types

### Habit Changes
- Use fixed conventional nomenclature for project naming
- Use capital letters for unit numbers
- Use abbreviations (Ave, St, etc.) at end of names

### Goals
1. Untangle information flow
2. Create clear set of data and relationships
3. Build more usable interface using Appsmith
