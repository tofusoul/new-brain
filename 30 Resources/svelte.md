# Routes
Any folder inside `src/routes` becomes the name of a page. A `+page.svelte` file inside that folder defines the page's HTML content.

- Tips

- DaisyUI and TailwindCSS are for fast prototyping with full on scope for easy customization later on in production without hassle. Trust me bro..

- (https://www.youtube.com/watch?v=mJgNSN5YAMI)

- Use TypeScript with and without Svelte. Haven\'t learnt it? Go do it. It\'s easy. And these days autocomplete will help you a lot. And you will see your code improve by a factor of 40% at least. Use @ts-ignore wherever you feel lost. It\'s fine!!

- If possible use SvelteKit for Authentication and Authorization API/purposes with Same-Site and HTTPOnly cookies options. Like the full backend frontend combo. For the rest of the API requests you can use the SvelteKit backend as a proxy to your original backend or use it as a backend itself. The advantage of it is that you don\'t have to worry about app security at all. You are 90% done with the security best practice part.
