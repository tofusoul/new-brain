## 💾 Important Notes

### Why Systems Die (AuDHD Pattern)
- Manual overhead → overwhelm → abandonment
- "Too broken to fix" → system collapse
- Perfectionism → never good enough → paralysis
- **The fix:** Design forgiving flows that survive when you're NOT maintaining them

### The Forgiving Flow Principle
> "Systems should work for you even when you ignore them for weeks or months."

**Implementation:**
- Daily tasks should be actionable in <2 minutes
- Weekly review: scan overdue, move to today if <5 min work
- Templates: simple, capture-focused, maintenance-light
- Fallbacks: When query breaks, manual capture still works
