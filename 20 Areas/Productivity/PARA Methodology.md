# PARA Methodology

**Type**: Resource  
**Created**: 2025-09-24  
**Last Updated**: 2025-09-24  

## Overview

PARA is a simple, comprehensive system for organizing digital information across any platform. Created by [[Tiago Forte]], it organizes information based on actionability rather than traditional subject-based categorization.

## The Four Categories

### 1. **P**rojects
Short-term efforts with specific goals and completion dates.

**Examples:**
- Complete webpage design
- Buy a new computer  
- Write research report
- Renovate bathroom
- Finish Spanish language course
- Set up new living room furniture

**Key characteristics:**
- Have a clear endpoint
- Are time-bound
- Drive immediate action
- Turn over regularly

### 2. **A**reas  
Ongoing responsibilities that require maintenance but have no end date.

**Examples:**
- Work: Marketing, Human Resources, Product Management, Engineering
- Personal: Health, Finances, Kids, Writing, Car, Home

**Key characteristics:**
- Continue indefinitely
- Require regular attention
- Have standards to maintain
- Don't have completion dates

### 3. **R**esources
Topics of ongoing interest and learning.

**Examples:**
- Graphic design
- Personal productivity
- Organic gardening
- Coffee
- Modern architecture
- Web design
- Japanese language
- Notetaking

**Key characteristics:**
- Interest-based, not responsibility-based
- Can be picked up and put down
- Support projects and areas
- Accumulate over time

### 4. **A**rchives
Inactive items from the other three categories.

**Examples:**
- Completed projects
- Inactive areas of responsibility
- Resources no longer interested in

**Key characteristics:**
- No longer active
- Saved for reference
- Can be safely ignored
- May be reactivated later

## Key Principles

### Organize by Actionability
Instead of organizing by broad subjects (like in school), organize by projects and goals you're committed to right now. This ensures all relevant material is together when you need it.

### Break Down Areas into Projects
Areas of responsibility can't be managed effectively until broken down into specific projects:
1. **Know the extent of commitments** - Projects make workload visible
2. **Connect efforts to long-term goals** - Projects create regular victories and motivation

### Benefits of PARA
- **Comprehensive**: Encompasses all information types
- **Flexible**: Adapts to different seasons of life
- **Cross-platform**: Works in any digital environment
- **Actionable**: Directly supports goal achievement
- **Simple**: Minimal maintenance overhead

## Implementation in Current Setup

### Directory Structure
```
~/notes/
├── Projects/          # Active projects with specific goals
├── Areas/            # Ongoing responsibilities  
├── Resources/         # Topics of interest
└── Archives/         # Inactive items
```

### Integration with Obsidian
- Use [[wiki links]] to connect related notes
- Tag notes with #Projects #Areas #Resources #Archives
- Create daily/weekly notes that reference PARA items
- Use Dataview or similar plugins for PARA management

## Best Practices

1. **Review Regularly**: Move completed projects to Archives, adjust areas as responsibilities change
2. **Keep it Current**: Only active items should be in Projects, Areas, Resources
3. **Link Liberally**: Use wiki links to show relationships between items
4. **Be Specific**: Projects should be concrete and actionable
5. **Trust the System**: Use PARA consistently across all digital tools
